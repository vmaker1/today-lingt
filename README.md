# 오늘의 빛 (Today's Light)

매일의 신앙 훈련 컴패니언 웹앱 — **프로토타입**입니다.
말씀 · QT · 기도 · 찬양 · 예배 · 실천 · 신앙계발의 일곱 훈련을 매일 점검하고,
신앙일기 · 나눔 · 방 · 쪽지 · 포인트 · 성령의 아홉 열매 도전까지 담았습니다.

React + Vite로 만들어졌고, 별도 서버 없이 그대로 웹에 올릴 수 있습니다.

---

## ⚠️ 먼저 알아두기 (지금 되는 것 / 안 되는 것)

이건 **화면과 흐름이 도는 프로토타입**입니다. 다음은 아직 "흉내"만 냅니다:

- 회원가입 · 본인인증 (실제 문자·이메일 인증 아님)
- 쪽지 · 방 나눔 (실시간 전송 아님, 내 화면에서만)
- 신앙일기 · 포인트 저장 (새로고침하면 초기화)
- 날씨 (오늘 값 고정)
- QT 유튜브 "매일 자동 갱신"

이 기능들이 **진짜로** 동작하려면 **서버(백엔드)** 가 필요합니다. → 아래 "다음 단계" 참고.
지금 배포로는 "폰에서 앱처럼 열어보고, 링크로 공유"하는 것까지 됩니다.

> QT의 유튜브 **재생목록**은 배포 후 앱 화면 안에서 실제로 재생됩니다.
> (지금 클로드 미리보기 안에서는 보안 제약으로 "새 탭 열기"로 넘어갈 수 있어요.)

## 💻 내 컴퓨터에서 실행해보기

1. **Node.js 설치** (한 번만) — https://nodejs.org 에서 LTS 버전 설치
2. 이 폴더에서 터미널을 열고:

```bash
npm install     # 처음 한 번, 필요한 것들 내려받기
npm run dev      # 개발 서버 실행
```

3. 터미널에 뜨는 주소(예: `http://localhost:5173`)를 브라우저에서 열면 끝.

배포용으로 정적 파일을 만들려면:

```bash
npm run build    # dist/ 폴더에 결과물 생성
npm run preview  # 만든 결과물 미리보기
```

## 🚀 웹에 올리기 (배포)

### 방법 A. Vercel + GitHub (추천 · 무료 · 자동)

1. https://github.com 에서 저장소(repository)를 새로 만들고, 이 폴더를 올립니다.
   - 터미널에서:
     ```bash
     git init
     git add .
     git commit -m "오늘의 빛 프로토타입"
     git branch -M main
     git remote add origin https://github.com/내아이디/저장소이름.git
     git push -u origin main
     ```
2. https://vercel.com 에 GitHub 계정으로 로그인 → **Add New → Project** → 방금 만든 저장소 선택.
3. Vercel이 Vite를 자동 인식합니다. 그대로 **Deploy**만 누르면 끝.
   - (혹시 물어보면: Framework `Vite`, Build `npm run build`, Output `dist`)
4. `https://...vercel.app` 주소가 생깁니다. 폰에서 열고 → 브라우저 공유 → **홈 화면에 추가**하면 앱 아이콘처럼 됩니다.
5. 앞으로 코드를 고쳐 `git push` 하면 **자동으로 다시 배포**됩니다.

### 방법 B. Netlify에 드래그 앤 드롭 (가장 간단 · GitHub 불필요)

1. 먼저 `npm run build`로 `dist/` 폴더를 만듭니다. (이 폴더엔 이미 `dist/`가 들어 있어요.)
2. https://app.netlify.com/drop 접속.
3. **`dist` 폴더를 통째로 화면에 끌어다 놓기**.
4. 바로 주소가 생깁니다. (수정하면 다시 build 해서 다시 드롭)

## 📁 폴더 구조

```
today-light/
├─ index.html          # 진입 페이지
├─ package.json        # 의존성·스크립트
├─ vite.config.js      # 빌드 설정
├─ dist/               # 빌드 결과물 (드래그 배포용)
└─ src/
   ├─ main.jsx         # React 시작점
   ├─ index.css        # 전역 스타일
   └─ App.jsx          # 앱 전체 (화면·기능)
```

수정은 대부분 `src/App.jsx` 안에서 합니다.
QT 유튜브 재생목록은 `App.jsx`의 `QT_LINKS` 부분을 고치면 됩니다.

## 🔜 다음 단계 — 서버(백엔드) 붙이기

프로토타입에서 진짜 서비스로 가려면 서버가 필요합니다. 크게 세 덩어리:

1. **계정 · 본인인증** — 이메일/휴대폰 인증(문자 API), 로그인 세션
2. **데이터 저장** — 신앙일기, 포인트, 방·쪽지, 성장 단계 (데이터베이스)
3. **매일 자동 갱신** — 유튜브 데이터 API로 "오늘의 큐티 영상"을 매일 자동으로 가져오기,
   오늘의 말씀·암송 구절·날씨도 매일 갱신

추천 조합(예): 프론트는 지금처럼 Vercel, 백엔드·DB·인증은 **Supabase** 또는 **Firebase**.
유튜브 자동 갱신은 **YouTube Data API v3**(무료 키) + 하루 1회 갱신하는 작은 서버 함수로 해결됩니다.
